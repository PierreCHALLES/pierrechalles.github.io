Introduction

Bienvenue sur le portfolio StructurAll. Cette page présente les détails du projet, ses évolutions techniques, et les résultats obtenus.

Compte Rendu StructurAll - Migration et Mise à Jour

Contexte :

Le projet StructurAll a nécessité une importante mise à jour pour aligner les technologies sur les standards modernes, améliorer les performances et garantir la pérennité du système. Voici les principales étapes et réalisations de ce projet.

1. Migration de Java 1.7 vers Java 1.17

  • Remplacement de XStream par Jakarta : La migration de Java a impliqué l’abandon de XStream au profit de Jakarta pour s’aligner sur les normes Jakarta EE modernes.
  • Adaptation du code existant : Les incompatibilités liées aux anciennes API ont été corrigées pour garantir le bon fonctionnement avec Java 17.

2. Mise à Jour des Librairies

  • Mises à jour des versions : Adoption des nouvelles versions des dépendances critiques, notamment LogManager 2.22 et Jersey 3.1.0.
  • Implémentation d’alternatives : Lorsque les librairies précédentes n’étaient plus supportées, des solutions alternatives ont été intégrées.

3. Migration de la Base de Données MongoDB

  • Passage de Mongo 2.6.0 à 6.0.18 : Une migration majeure a permis d'exploiter les fonctionnalités avancées de MongoDB 6.0.18.
  • Mise à jour des fichiers Mongo :
    • ContenuMongo et StructureMongo ont été adaptés pour garantir une interaction optimale avec la nouvelle version de MongoDB.
    • Les appels aux collections ont été revus pour maintenir la compatibilité avec le nouveau pilote MongoDB.

4. Gestion des Conversions Objets/Xml

  • Fichiers de conversion (FicTools) : Ces fichiers ont été modifiés pour garantir une conversion fluide des objets Java en XML tout en respectant les nouvelles normes Jakarta.

5. Migration du Serveur

  • Passage de Tomcat 7 à Tomcat 10.1.29 :
    • Le serveur d’applications a été mis à jour pour bénéficier des optimisations et fonctionnalités offertes par la dernière version de Tomcat.
    • Les fichiers de configuration XML du serveur ont été modifiés pour refléter les nouvelles dépendances et configurations requises.
  • Ajout des nouvelles librairies : Les librairies nécessaires (notamment Jakarta et MongoDB) ont été intégrées au serveur.

6. Gestion de la Structure et du Code Source

  • Structuration GitLab :
    • Chaque étape de la migration a été versionnée sur GitLab.
    • Un suivi rigoureux des commits a permis de documenter chaque changement et de garantir la traçabilité.
